Introduction

ORCCAD V4 results from several years of work by INRIA teams. It relies on well-established concepts in robotics, automatic control and computer science. It is designed for ensuring modularity and interoperability.

ORCCAD V4 allows to define elementary actions (Robot-Task) using block-diagram oriented GUI and their composition (Robot-Procedure) using the Esterel language. It includes the major part of the ORCCAD features which are described in the papers. It allows safe installing and running for a reasonably wide class of real-time applications.

There are two kind of code controller :

  • code C++ generated is a single rate, single processor for hard real-time (VxWorks) or soft real-time (Linux 2.6, Windows XP)
  • code C generated is multiple rate, distributed for hard real-time (VxWorks, RTAI/Linux 2.6)

The distribution of the software will be Open Source (see the definition at http://www.opensource.org/docs/definition.php). Therefore, standards development tools must be used.

One goal is to modelize complex robotics controller using the ORCCAD methodology

What are the documentation requirements?

The documentation must contain :

  • DOC-1.00 : a Software User Manual
  • DOC-1.01 : a Getting Started Document
  • DOC-1.02 : a Tutorial Document
  • DOC-1.03 : a Methodology Document
  • DOC-1.04 : a Developer Document

The Documentation format requirements are:

  • DOC-2.00 : The source of documents must be available in HTML format.
  • DOC-2.01 : All documents must be available in PDF format.
  • DOC-2.02 : All documentation must be written in English.

Environmental Requirements

What are the Data representation requirements?

  • F-2.300 XML format

What are the resource requirements?

RES-00: The recommended configuration is : processor 800 Mhz, 512 Mo Ram, 200 Mo of free space on HDD

What are the portability requirements?

Orccad V4 should run on :
  • POR-1.00: PC / Linux environment from kernel2.6
  • POR-1.01: PC / Windows environment Microsoft Windows XP

The data must be portable:

  • POR-2.00 : ASCII data files must be defined with an unique charset.
  • POR-2.01 : BINARY data files must be defined with an IEEE format norm.

The code controller generated must run on :

  • POR-3.00 : VxWorks hard real-time operating system.
  • POR-3.01 : RTAI/Linux 2.6 hard real-time operating system.
  • POR-3.02 : Linux 2.6 soft real-time operating system.
  • POR-3.03 : Windows XP soft real-time operating system.
